# Knowbase — AI-Powered Personal Knowledge Base

Knowbase is a self-hosted personal knowledge management system that automatically enriches everything you save. Paste a YouTube URL, a website link, or type a note — Knowbase fetches the content, extracts transcripts where available, summarises it, assigns tags, and groups it into collections. All AI work happens through a pluggable provider (OpenAI, Mistral, or a fully local Ollama model). No data leaves your machine if you use the local option.

---

## What It Does

| Input | What Knowbase does |
|---|---|
| **YouTube URL** | Extracts the full transcript, summarises it with key points and takeaways, generates tags and a collection |
| **Website link** | Fetches the full page content, summarises it, tags it |
| **Note / text** | Runs the same AI pipeline on your free-form text |
| **PDF upload** | Extracts text from the PDF and enriches it inline |

Every saved item gets:

- **AI summary** — structured as Overview, Key Points, and Takeaways (rendered as markdown)
- **Tags** — 3–8 topic tags auto-generated by AI
- **Collection** — assigned to a thematic group (AI creates new collections when needed)
- **AI-generated title** — if you leave the title blank
- **Related items** — items that share tags surface as related content

You can optionally guide the AI with:
- **Additional context** — background information the AI should know
- **AI instructions** — e.g. "Summarise for a beginner" or "Focus on security implications"

## Architecture

```
┌─────────────────────────────────────────────────────────────────────┐
│  Browser  →  Next.js (port 3000)                                    │
│               │                                                      │
│               │ REST API                                             │
│               ↓                                                      │
│  Flask API (port 5001)                                              │
│     │  saves item (status: pending)                                 │
│     │  triggers n8n webhook                                         │
│     ↓                                                               │
│  n8n (port 5678)                                                    │
│     │  Content Ingestion workflow fires                             │
│     │  signs payload with HMAC-SHA256                               │
│     │  POSTs back to Flask /api/webhooks/n8n                       │
│     ↓                                                               │
│  Flask webhook handler                                              │
│     │  verifies HMAC signature                                      │
│     │  fetches YouTube transcript / webpage                        │
│     │  runs AI enrichment pipeline                                  │
│     │    ├── summarize()                                            │
│     │    ├── extract_tags()                                         │
│     │    ├── suggest_collection()                                   │
│     │    └── suggest_title()                                        │
│     │  saves results → status: enriched                            │
│     ↓                                                               │
│  Frontend polls every 3s until enriched, then shows results        │
└─────────────────────────────────────────────────────────────────────┘

Supporting services:
  PostgreSQL (port 5432)  — all persistent data
  Mailpit   (port 8025)   — catches all outbound email (dev)
  Ollama    (port 11434)  — local LLM inference (optional)
```

## Quick Start

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux)
- `make` (comes with Xcode CLI tools on Mac; `apt install make` on Linux)
- Git

### 1. Clone and configure

```bash
git clone https://github.com/Tanmoy2099/knowbase-demo-project.git
cd knowbase-demo-project
cp .env.example .env
```

Open `.env` and fill in the required values (marked `# REQUIRED`):

```env
SECRET_KEY=your-random-secret-key          # any random string
POSTGRES_PASSWORD=your-db-password
N8N_WEBHOOK_SECRET=your-webhook-secret     # any random string
N8N_ENCRYPTION_KEY=exactly-32-chars-here!!
```

### 2. Choose your AI provider and start

**Option A — Local (free, private, no API key)**

```bash
# Pull a model first (run once on the host, not in Docker)
ollama pull llama3.2

# Start everything
make up
```

Set in `.env`:
```env
AI_PROVIDER=ollama
OLLAMA_BASE_URL=http://host.docker.internal:11434
OLLAMA_MODEL=llama3.2
```

**Option B — OpenAI**

```bash
make up
```

Set in `.env`:
```env
AI_PROVIDER=openai
OPENAI_API_KEY=sk-...
```

**Option C — Mistral**

```bash
make up
```

Set in `.env`:
```env
AI_PROVIDER=mistral
MISTRAL_API_KEY=...
```

### 3. Run migrations and sync workflows

```bash
make migrate          # applies database schema
make sync-workflows   # pushes n8n workflow definitions
```

### 4. Set up n8n (first run only)

1. Open [http://localhost:5678](http://localhost:5678)
2. Create an admin account (e.g. `admin@knowbase.local` / any password)
3. Generate an API key: **Settings → API → Create API Key**
4. Paste the key into `.env`:
   ```env
   N8N_API_KEY=eyJhbGc...
   ```
5. Restart the backend so it picks up the key and syncs workflows:
   ```bash
   docker compose restart backend
   ```

### 5. Open the app

| Service | URL |
|---|---|
| **Knowbase UI** | http://localhost:3000 |
| **n8n workflow editor** | http://localhost:5678 |
| **Mailpit email viewer** | http://localhost:8025 |
| **Flask API** | http://localhost:5001 |

## AI Provider Setup Details

### Local Ollama (recommended for privacy)

Ollama must be installed and running **on your host machine** (not inside Docker). The backend container reaches it via `host.docker.internal`.

```bash
# Install: https://ollama.com
ollama pull llama3.2          # 2 GB — default, fast
ollama pull llama3.1:8b       # 4.7 GB — better quality
ollama pull mistral           # 4.1 GB — Mistral-based local

# Set the model in .env
OLLAMA_MODEL=llama3.2
```

Enrichment with a local model takes 30–120 seconds per item depending on your hardware.

> **macOS note:** Port 5000 is taken by AirPlay Receiver. Knowbase already maps the backend to **5001** to avoid this conflict.

### OpenAI

Requires an OpenAI account with billing enabled.

```env
AI_PROVIDER=openai
OPENAI_API_KEY=sk-proj-...
```

Uses `gpt-4o-mini` for all operations (fast and cheap). Change `fast_model` in `backend/app/ai/providers/openai_provider.py` to `gpt-4o` for higher quality.

### Mistral

Requires a [Mistral AI](https://mistral.ai) account.

```env
AI_PROVIDER=mistral
MISTRAL_API_KEY=...
```

Uses `mistral-small-latest`.

## Data Flow Details

### Saving content

```
User pastes URL or text → type auto-detected client-side
  youtube.com/* / youtu.be/*  →  type: youtube
  http(s)://*                 →  type: link
  anything else               →  type: note

POST /api/content with { type, raw_url / body, title?, extra_context?, user_instructions? }
  Flask saves item (status: pending)
  Flask triggers n8n webhook (content-ingestion)
  Returns immediately with item ID

n8n Content Ingestion workflow:
  Signs payload (HMAC-SHA256)
  POSTs signed payload back to Flask /api/webhooks/n8n

Flask webhook handler:
  Verifies HMAC signature (401 if invalid)
  For youtube → YouTubeTranscriptApi.get_transcript() + oEmbed metadata
  For link    → fetches full page HTML (up to 50 KB)
  For note    → content is already the body
  Calls AI enrichment pipeline:
    summarize()         → Overview + Key Points + Takeaways (markdown)
    extract_tags()      → 3–8 topic tags
    suggest_collection()→ existing or new collection (confidence ≥ 0.7)
    suggest_title()     → auto-title if user left blank
  Saves everything → status: enriched

Frontend polls GET /api/content/:id every 3s
  Spinner disappears when status = enriched
```

### n8n Workflows

| Workflow | Trigger | Purpose |
|---|---|---|
| **Content Ingestion** | Webhook POST | Routes content save through the signing pipeline |
| **AI Enrichment Callback** | Sub-workflow | Called to POST enriched data back to Flask |
| **YouTube Transcript** | Called inline | Fetches transcript + oEmbed metadata |
| **Webpage Fetch** | Called inline | Fetches link content |
| **Scheduled Graph Update** | Cron 2am daily | Rebuilds the topic relationship graph |
| **Retry Stuck Items** | Cron every 10 min | Re-triggers items stuck in pending > 10 min |

Workflows are defined as JSON in `backend/app/workflows/` and synced idempotently to n8n on every backend startup via `WorkflowSyncService` (SHA-256 hash comparison — only changed workflows are updated).

### HMAC webhook security

n8n and Flask share a secret (`N8N_WEBHOOK_SECRET`). Flask pre-computes the HMAC when triggering n8n, embeds it in the trigger payload, and n8n echoes it back as the `X-N8N-Signature` header. Flask verifies it before processing any enrichment callback. Requests without a valid signature get a `401`.

## Usage Guide

### Saving content

The save form auto-detects what you pasted:

- Paste a **YouTube URL** → badge shows "▶ YouTube"
- Paste any **https://** link → badge shows "🔗 Link"
- Type anything else → badge shows "📝 Note"

Click **More options** to add:
- **Title** — overrides the AI-generated title
- **Additional context** — background information for the AI
- **AI instructions** — e.g. "Focus on the business implications" or "Explain for a 10-year-old"

### PDF upload

Use the separate PDF upload section. Text is extracted locally (no third-party OCR), then enriched with the same AI pipeline.

### Filtering and browsing

- **Collections sidebar** — click to filter by collection
- **Tags sidebar** — click to filter by tag
- **Items expand** on click to show summary, tags, collection, and an "Open in new tab" button
- Tags and collections with zero items are automatically cleaned up

### Admin endpoints

```
GET  /api/admin/health           — service health check
POST /api/admin/sync-workflows   — re-sync n8n workflows
POST /api/admin/rebuild-graph    — rebuild topic relationship graph
GET  /api/admin/stuck-items      — list items stuck in pending > 10 min
POST /api/admin/retry-item/:id   — manually re-trigger a stuck item
```

### Adding a new AI provider

1. Create `backend/app/ai/providers/your_provider.py` implementing `AIProvider`:
   ```python
   class YourProvider(AIProvider):
       def summarize(self, content, context) -> SummaryResult: ...
       def extract_tags(self, content) -> list[TagResult]: ...
       def suggest_collection(self, content, existing) -> CollectionSuggestion | None: ...
       def suggest_title(self, content, content_type) -> TitleResult: ...
   ```
2. Register it in `backend/app/ai/factory.py`:
   ```python
   _REGISTRY = {
       "openai":  OpenAIProvider,
       "mistral": MistralProvider,
       "ollama":  OllamaProvider,
       "yours":   YourProvider,   # ← add here
   }
   ```
3. Set `AI_PROVIDER=yours` in `.env`.

## Troubleshooting

**Items stuck in "Pending…" forever**
- Check backend logs: `make logs-backend`
- Verify n8n is healthy: `curl http://localhost:5678/healthz`
- Check the n8n UI for failed workflow executions: http://localhost:5678
- The retry cron fires every 10 minutes automatically
- Manual retry: `curl -X POST http://localhost:5001/api/admin/retry-item/<id>`

**YouTube transcripts not extracting**
- Not all videos have transcripts enabled (auto-captions or manual)
- Geo-restricted videos may not have transcripts available externally
- The pipeline falls back to oEmbed metadata and still produces a summary

**Ollama enrichment is slow**
- Expected: 30–120s depending on model size and hardware
- `llama3.2` (2 GB) is fastest; `llama3.1:8b` is better quality but slower
- GPU acceleration: ensure Ollama is using your GPU (`ollama ps` to check)

**"Failed to fetch" errors in browser**
- Confirm the backend is running: `docker ps`
- Check NEXT_PUBLIC_API_URL in `.env` matches the backend port (5001)
- On Mac: port 5000 is taken by AirPlay — Knowbase uses 5001 by design

**n8n workflow sync fails on startup**
- Ensure `N8N_API_KEY` is set in `.env`
- The API key must be created in the n8n UI first (Settings → API)
- If n8n isn't ready yet, sync is retried automatically on next restart

## Database Schema

```
content_items          — core entity (type, status, raw_url, body, title, extra_context, user_instructions)
  └─< content_tags >─ tags              — many-to-many tag associations
  └─< collection_items >─ collections   — many-to-many collection membership
  └─< summaries                         — AI-generated summaries (one per item)
  └─< topic_relations                   — graph edges (Jaccard similarity on shared tags)
workflow_sync                           — tracks n8n workflow sync state (SHA-256 hashes)
```

Migrations are managed with Alembic. Never edit migration files after they have been applied.
